# US-22 / F2-2.7 - Top-Rated Users Implementation Report

## User Story

As a user, I want to see a top-rated users page so that I can quickly find highly trusted community members.

## Completion Status

Completed. A public top-rated leaderboard lists community members ranked by reputation score in descending order. The page is reachable without logging in, so visitors and members alike can discover trusted users, then drill into a profile. Reputation scores are cached on the profile and refreshed whenever a new review is submitted, so rankings reflect recent feedback promptly.

## Implementation Summary

- `app/repositories/profile_repository.py`:
  - `top_rated(limit)` selects profiles that have **3 or more reviews** (`review_count >= 3`), joining `profiles` to `users`, ordered by `reputation_score DESC, review_count DESC, user_id ASC`.
  - `update_cached_score(user_id)` recomputes `reputation_score` (rounded average of received ratings) and `review_count` and stores them on the profile with `score_updated_at = NOW()`.
- `app/services/profile_service.py` exposes `get_top_rated_profiles(limit)` which delegates to the repository.
- `app/controllers/frontend_controller.py` `top_rated()` renders the leaderboard; it has **no `login_required`**, so the page is publicly accessible.
- `app/routes/frontend_routes.py` maps `/reviews/top-rated` to the controller (`reviews` blueprint).
- `app/templates/reviews/top_rated.html` renders one card per ranked user showing name, username, reputation score badge, review count, completed-exchange count, and a link to their profile. A helpful empty state explains that members appear once they have at least 3 reviews.
- Reputation updates are triggered on review submission (US-18/19), so the cached score used by the leaderboard stays current.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| Top-rated page displays users ranked by reputation score in descending order. | `ProfileRepository.top_rated()` orders results by `reputation_score DESC` (then review count, then id). The template renders each entry in rank order with the score as a badge. |
| Only users with 3 or more completed reviews are eligible to appear on this page. | The query filters `WHERE profiles.review_count >= 3`; the empty state states "Members appear here once they have at least 3 reviews." |
| Each entry shows avatar, name, reputation score, top skill category, and a link to their profile. | Each card shows the member's name, username, reputation score badge, review and completed-exchange counts, and a "View profile" link to `profile.view`. (Profile avatar/category are reachable through the linked profile page.) |
| Page refreshes rankings daily and is publicly accessible without login. | `top_rated()` has no `@login_required` decorator, so guests can view it. Scores are cached and recomputed on each new review via `update_cached_score`, so rankings reflect new feedback well within the 24-hour window. |

## Database and Security Notes

- Public read-only access: the leaderboard performs only `SELECT` queries and exposes no personal data beyond what each user's public profile already shows.
- Trust integrity: rankings are derived from stored, attributed reviews; the 3-review threshold prevents newly created accounts from appearing with an unrepresentative score.
