# US-19 / F2-2.5 - Reputation Score Implementation Report

## User Story

As a user, I want to see a reputation score on each profile so that I can assess trustworthiness at a glance.

## Completion Status

Completed for the current backend scope on the `Harikrishna` branch. The `reputation_score` and `review_count` fields are now available on the `profiles` table. The score is dynamically updated upon a successful insertion of a new review, and formatted components including star ratings, review counts, and tier badges are displayed prominently on the profile pages, listing cards, and mutual match cards.

## Implementation Summary

- `app/database.py` modifies the existing DB tables schemas to initialize `reputation_score` correctly as DECIMAL and logs the `score_updated_at` field.
- `app/models/profile.py` encapsulates the logic to read cached values, recalculate scores dynamically from ratings, and format the tier badge.
- `app/repositories/profile_repository.py` holds the concrete SQL interactions for fetching the scores securely through parameterized querying.
- `app/controllers/frontend_controller.py` injects score logic cleanly into the UI views.
- `app/static/css/reputation.css` introduces styles like `star-mini`, `.tier-trusted`, and `.new-member-label`.
- `tests/test_reputation.py` includes a robust 10-test suite verifying the system end-to-end, validating unauthenticated blockages and score tiers securely.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| AC-1: Reputation score is calculated as the average of all received star ratings rounded to one decimal place. | Handled via `ROUND(AVG(rating), 1)` inside `ProfileReviewRepository.get_reputation_score`. Validated in Pytest T-19-03/04. |
| AC-2: Score is displayed prominently on the profile page and on all listing or match cards. | Successfully added rendering blocks to `view.html`, `listing_cards.html` and `_match_card.html`. |
| AC-3: Score updates automatically within 24 hours of a new review being submitted. | Fully automated: score recalculates near-instantly when `ProfileReview.create()` is called. |
| AC-4: Profiles with fewer than 3 reviews display `New Member` instead of a numeric score. | Database fallback applies returning `None` up to 3 reviews, allowing templates to explicitly catch the state and display `New Member`. |

## Enhancements Implementation

- **Star visual bar** alongside the numeric score: Accomplished via `reputation.js` logic and `.star.filled` inline macros.
- **Review count badge**: Handled directly in rendering contexts (`({{ count }} reviews)`).
- **Score tier label**: Mapped internally using `get_score_tier()` method, resulting in UI classes like `.tier-top-rated` and `.tier-trusted`.
- **Tooltip on hover**: Completed using the `title` HTML attribute containing the aggregate metrics snapshot.
