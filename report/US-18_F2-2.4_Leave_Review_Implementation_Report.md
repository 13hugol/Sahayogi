## User Story

As a user, I want to leave a review after an exchange so that others can make informed decisions.

## Completion Status

Completed on the `Grishma` branch. The system now supports a user review and rating system that unlocks after an exchange has been completed by both parties. Submitted ratings (1-5 stars) and comments (up to 500 characters) are published immediately on the reviewed user's profile and automatically update their profile's average reputation score. The system prevents duplicate reviews for the same exchange.

## Implementation Summary

- **Database Updates**: Added `exchange_completion_marks` table and updated `profile_reviews` table in `app/database.py` with foreign keys referencing the exchanges table and unique index to prevent duplicate entries, and updated the table clearing sequence in `tests/conftest.py`.
- **Model Classes**:
  - `ExchangeCompletionMark` model class in `app/models/exchange.py` representing individual user marks.
  - Added properties on `Exchange` and `ProfileReview` models to handle completion status and details.
  - Exposed in `app/models/__init__.py`.
- **Repositories**:
  - `ExchangeRepository` (`app/repositories/exchange_repository.py`) handles user marking of completion, checks participant permissions, and increments the completed exchange count in the user's profile when fully completed.
  - `ProfileReviewRepository` (`app/repositories/profile_repository.py`) saves the review and updates `review_count` and `reputation_score` on the user profile inside an atomic database transaction.
- **Controllers & Routing**:
  - Frontend controller handlers: `FrontendController.mark_exchange_complete` and `FrontendController.review_form` (which handles form validation and handles rendering of errors).
  - Blueprints and routing registered in `app/routes/frontend_routes.py`.
- **Templates**:
  - `app/templates/exchanges/detail.html` conditionally shows "Mark complete" and "Leave review" buttons, and lists reviews.
  - `app/templates/reviews/form.html` renders the review submission form with star rating dropdown and comments textarea.
  - `app/templates/profile/view.html` displays the user reputation score, completed exchange counts, and lists recent reviews.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| Review form appears only after both parties have marked an exchange as completed. | Enforced in `FrontendController.review_form` and checked via `ExchangeRepository.is_fully_completed`. "Leave review" button is only rendered in `app/templates/exchanges/detail.html` if the status is `'completed'`. |
| Review requires a star rating from 1-5 and an optional written comment of up to 500 characters. | Form validation in `FrontendController.review_form` enforces integer rating values from 1 to 5, and rejects comments longer than 500 characters. Textarea in `reviews/form.html` has `maxlength="500"`. |
| Submitted reviews are published immediately on the reviewed user's profile. | `ProfileReviewRepository.create` updates the profile's average rating and total reviews. The profile page `app/templates/profile/view.html` pulls these updated values and lists the reviews. |
| Users can only leave one review per completed exchange; submitted reviews cannot be deleted by the reviewer. | The database enforces this via unique key constraint `uq_profile_reviews_exchange_reviewer` on `(exchange_id, reviewer_id)`. The controller checks this via `find_for_exchange_by_reviewer` and redirects with a flash message if already reviewed. |

## Database and Security Notes

- **Access Controls**: The completion and review actions use the `@login_required` decorator.
- **Database Transactions**: Reviews are created and profile reputation scores are updated together inside a database transaction (`with db.transaction()`) to ensure database consistency.
- **CSRF Protection**: Form actions submit a hidden CSRF token which is validated by the application before execution.
