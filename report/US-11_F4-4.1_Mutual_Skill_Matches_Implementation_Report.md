## Implementation Summary

- `ProfileRepository.get_mutual_matches()` evaluates active users across the platform returning matching user arrays and overlap quantities dynamically sorted by reputation and overlaps.
- `app/controllers/frontend_controller.py` orchestrates match retrieval (`matches()`) securely under `@login_required` decorators.
- Saving profiles from `/profile/edit` invokes `_notify_new_matches()` automatically, executing logic in `Notification` to trigger new match notifications for both peers if not already notified.
- `Notification` model handles `type` and `target_url` persistence through the updated schema in `database.py`.

## Database and Audit Notes

- Changes applied to `notifications` table schema: added `type` (e.g. `new_match`) and `target_url` for deep-linking.
- Excluded users with `banned` status from match listings entirely.
- Added overlap caching detection via `target_url` parsing to avoid repetitive notification generation if duplicates occur.
