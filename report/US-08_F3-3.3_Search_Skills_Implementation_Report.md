As a user, I want to search skills by keyword so that I can quickly find specific expertise I need.

## Completion Status

Implemented on the `Chayanga` branch for the current Flask/Jinja backend scope. The existing listings page and live-search JavaScript now connect to a search-backed service that returns approved skill listing results by keyword.

## Implementation Summary

- Added `SkillSearchListing` as the view model used by listing cards and listing detail pages.
- Added `SkillSearchRepository` with parameterized SQL for approved skill search.
- Added `SkillSearchService` for keyword cleanup, pagination, total count calculation, and listing lookup.
- Added the `skill_search_listings` table used by the search story.
- Added seed examples for the search index when the table is empty, including `Guitar lessons for beginners` and `Guitarist jam and stage confidence` so partial keyword search can be demonstrated.
- Updated `FrontendController.marketplace()` to render search results for submitted `q` values.
- Updated `FrontendController.api_search()` so the existing live-search JavaScript receives the matching HTML and result count.
- Updated listing detail rendering so search results without a linked profile still display correctly.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| Search bar is visible on the browse page and returns results matching title or description keywords. | `app/templates/listings/index.html` already provides the keyword field; `SkillSearchRepository.search()` matches `title LIKE %keyword% OR description LIKE %keyword%`. |
| Search results update dynamically or on submit and display the number of results found. | `app/static/js/app.js` posts the form query to `/listings/api/search`; `api_search()` returns `{ count, html }`, and the page updates `#resultCount` and `#listingResults`. |
| Partial keyword matching is supported. | SQL uses `%keyword%` patterns; focused tests verify `guitar` returns both `Guitar lessons` and `Guitarist` results. |
| If no results are found, a friendly message is shown. | `app/templates/partials/listing_cards.html` shows `No listings found` with guidance to broaden keywords or remove filters. |

## Database and Security Notes

- Search queries use parameterized SQL placeholders.
- Only approved rows from `skill_search_listings` are returned.
- The implementation does not add category filtering, location filtering, listing creation, request handling, or admin moderation changes.
- The seed examples are inserted only when the search table is empty.
