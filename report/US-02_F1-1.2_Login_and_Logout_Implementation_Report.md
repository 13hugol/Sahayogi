## Implementation Summary

- `app/controllers/auth_controller.py` handles `/auth/login` and `/auth/logout`.
- `User.find_by_email()` loads users by email.
- `User.check_password()` verifies submitted passwords against the stored password hash.
- Failed login attempts increment `failed_login_count`.
- After the configured threshold is reached, `locked_until` is set for the lockout period.
- `login_user()` establishes the Flask-Login session after successful authentication.
- `logout_user()` invalidates the session.
- The authenticated application layout exposes logout from the top bar.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| Login accepts valid email and password. | `AuthController.login()` loads the user by email and checks the password hash before calling `login_user()`. |
| Incorrect credentials display a clear error. | Invalid email or password flashes `Invalid email or password.` and re-renders the login page. |
| Session is maintained across pages. | Flask-Login is initialized in `app/__init__.py`, and protected pages use `login_required`. |
| Logout is accessible and invalidates the session. | `/auth/logout` calls `logout_user()` and redirects to the public home page. |
| Three failed attempts lock the account for 10 minutes. | The controller applies `LOCKOUT_THRESHOLD` and `LOCKOUT_DURATION_MINUTES`, storing lockout state on the user record. |

## Database and Security Notes

- Authentication data is stored in the `users` table.
- Password verification uses the secure hash rather than comparing plain text.
- Session access is centralized through Flask-Login.
- Lockout state is persisted in MySQL, so it survives page refreshes and new requests.
