# US-15 / F4-4.5 - Unread Message Badge Implementation Report

## User Story

As a user, I want to see an unread message badge so that I know when I have new messages waiting.

## Completion Status

Completed on the `Aayan` branch. The system now supports a dynamic, real-time unread messages badge on the sidenav "Messages" item. The badge is hidden on initial load if the user has 0 unread messages, updates asynchronously using the `/notifications/counts` endpoint, and decrements or disappears as conversations are opened and read.

## Implementation Summary

- **Backend Context Processor**: Updated the global template context processor in `app/__init__.py` to fetch the actual count of unread messages for the currently logged-in user using `MessageRepository().count_unread(current_user.id)` and populate `nav_counts.messages`.
- **Backend JSON Endpoint**: Modified the `/notifications/counts` endpoint in `app/controllers/frontend_controller.py` to retrieve and return the real `unread_messages` count (under the key `messages`) instead of a hardcoded value.
- **Frontend Template Integration**: Added the conditional `d-none` class to the messages unread badge (`<b class="nav-count sidenav-count{% if nav_counts.messages == 0 %} d-none{% endif %}" data-count-type="messages">`) in `app/templates/base.html` so it is hidden on initial load when the count is 0.
- **Real-Time Polling Logic**: The existing JavaScript in `app/static/js/app.js` queries `/notifications/counts` periodically and updates the `.nav-count` text content, toggling the `d-none` class dynamically depending on whether the count is greater than zero.
- **Conversation Auto-Read**: Opening a conversation details page `/messages/<conversation_id>` triggers marking all posts in that conversation as read, thereby updating the unread count database-side.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| A numeric badge appears on the Messages icon in the navigation bar showing the count of unread messages. | Rendered in `app/templates/base.html` using the template variable `{{ nav_counts.messages }}`. |
| Badge count updates in real-time (or on page refresh) without requiring a full page reload. | Asynchronous calls to `/notifications/counts` retrieve the live count, which the script in `app/static/js/app.js` updates in the DOM. |
| Opening a conversation marks all messages in that thread as read and decrements the badge count. | Navigating to `/messages/<conversation_id>` invokes `MessageService.get_conversation_posts`, which updates post read statuses and triggers a decrement of the user's unread count. |
| Badge disappears entirely when there are zero unread messages. | Conditional rendering inside `app/templates/base.html` adds the `d-none` bootstrap class when count is 0, and the javascript polling checks if count is 0 to toggle `d-none`. |

## Database and Security Notes

- **Repository Counts**: The unread count is computed using a fast SQL count query (`SELECT COUNT(*) FROM message_posts p JOIN message_participants mp ON p.conversation_id = mp.conversation_id WHERE mp.user_id = %s AND p.sender_id != mp.user_id AND p.read_at IS NULL`).
- **Access Control**: Users can only fetch counts for their own accounts, verified through the `current_user.id` session variable.
