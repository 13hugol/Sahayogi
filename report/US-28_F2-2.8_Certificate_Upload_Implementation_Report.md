## Completion Status

Completed on the `Grishma` branch. The system now supports certificate file upload (PDF, PNG, JPG, JPEG up to 10MB) during listing creation/editing and profile management, admin moderation queue for review/approval/rejection of uploaded certificates, audit logging, in-app notifications, and verified badge display across listings and user profiles.

## Implementation Summary

- **Database updates**: Used existing schema structures (`profile_certificates` and `skills` table columns `certificate_path` and `certificate_status`) to track certificate state.
- **Model updates**: Added the `user` property to `ProfileCertificate` in `app/models/profile.py` to fetch user details for moderation.
- **Repository updates**: Added find, upsert, status updates, and deletion helpers to `ProfileCertificateRepository` in `app/repositories/profile_repository.py` and `SkillRepository` in `app/repositories/skill_repository.py`. Also added helper method `ensure` to `CategoryRepository` in `app/repositories/category_repository.py` to fix test seeder setups.
- **Service updates**: Added `notify_certificate_approved` and `notify_certificate_rejected` to `NotificationService` in `app/services/notification_service.py`.
- **Controller updates**:
  - Implemented form processing, validations, replacement, and removal of certificates in `post_listing` of `FrontendController` (`app/controllers/frontend_controller.py`).
  - Implemented the certificates queue listing and file upload processing in `certificates` of `FrontendController`.
  - Implemented approval (`approve_certificate`) and rejection (`reject_certificate`) actions in `AdminController` (`app/controllers/admin_controller.py`), writing audit logs and sending notifications.
- **Routes Integration**: Registered routes for certificate approval and rejection in `app/routes/admin_routes.py`.
- **UI / Template Updates**:
  - Updated `listings/form.html` to support `enctype="multipart/form-data"`, file input, and removal checkbox.
  - Updated `listings/detail.html` and `partials/listing_cards.html` to display the `Verification Pending` badge when the listing's certificate is pending review.

## Acceptance Criteria Evidence

| Acceptance Criterion | Implementation Evidence |
| --- | --- |
| Skill listing form includes an optional certificate upload field accepting PDF or image files up to 10MB. | The create/edit listing form includes a file input field that accepts PDF, JPG, JPEG, and PNG. Server-side validation rejects empty, oversized (>10MB), or unsupported file types. |
| Uploaded certificates are sent to admin for manual review; listing shows 'Verification Pending' badge until approved. | Uploading a certificate sets the listing's status to `pending` review. A yellow `Verification Pending` badge is displayed on listing cards and details. |
| Approved certificates display a 'Verified' badge on the listing and the user's profile. | When approved by an admin, the badge changes to a teal `Verified` badge on the listing, and a `Verified certificate` badge is shown on the owner's profile page. |
| Users can replace or remove a certificate from their listing at any time, which resets verification status. | The edit listing form has options to replace (upload new) or remove (checkbox check). Doing so updates or deletes the certificate record and resets the status to `none` or `pending`. |

## Database and Security Notes

- **Upload Folder**: Certificates are securely saved inside the configured upload folder under `certificates/` directory using unique UUID filenames.
- **MIME & Header Validation**: Validates both the file extension and the internal byte headers (e.g. `%PDF` for PDF, PNG signature for PNG, JPEG markers for JPG) to prevent execution or upload of spoofed files.
- **Audit Logs**: All admin certificate moderation actions (approvals and rejections) are logged inside the database `admin_audit_logs` table.
