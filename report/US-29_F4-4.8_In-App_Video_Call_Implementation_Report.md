## Implementation Summary

- `app/database.py` adds table column definitions and migration operations for user activity tracking and active video call fields.
- `app/models/user.py` adds `last_active_at` attribute and `is_online` property returning true if user made any request within 2 minutes.
- `app/models/exchange.py` updates to include call state indicators: `video_call_active`, `video_call_started_at`, and `video_call_ended_at`.
- `app/repositories/exchange_repository.py` handles starting call, ending call, and fetching active exchange between two users.
- `app/controllers/frontend_controller.py`:
  - `video_call_room()` renders the live video room.
  - `start_video_call()` sets call state in database and posts a system message with a Join Call button.
  - `end_video_call()` terminates the call, computes elapsed duration, and logs a summary to `video_session_summary`.
  - `video_call_heartbeat()` polls peer online status, updates user presence, and returns room state.
- `app/routes/frontend_routes.py` maps video call endpoints under the `/exchanges` url prefix.
- `app/templates/exchanges/video.html` implements webcam rendering, audio/video toggle states, and 2-minute rejoin countdown timers.
- `app/templates/messages/detail.html` includes start call controls (disabled if peer is offline, active if online, joinable if active).

## Database and Security Notes

- Auth Protection: All video call endpoints (`video_call_room`, `start_video_call`, etc.) check if the logged-in user is one of the exchange participants (learner or teacher). If not, it aborts with a `403 Forbidden` response.
- Status Protection: Calls can only be initiated or conducted on exchanges that have an `active` status.
